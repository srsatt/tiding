# Project Backlog

This backlog captures user feedback, corrections, and follow-up tasks. Every
new user-raised point must be added here unless it is fully completed in the
same change and recorded with evidence.

## Policy

- Add every user point with status, area, source/date, and completion evidence,
  including corrections to agent prompts, quality gates, UX expectations, and
  implementation approach.
- Split broad feedback into concrete implementation or documentation tasks.
- Move an item to Done only when code, docs, tests, or browser evidence prove
  completion.
- Move an item to Deferred only with explicit user approval.
- Final project completion requires a clear backlog: no Open or In Progress
  items, and all user-raised points Done with evidence or Deferred with explicit
  user approval.
